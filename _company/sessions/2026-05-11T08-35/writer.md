요청하신 대로 기획-스크립트-디자인-업로드 단계별로 에이전트 간의 데이터 포맷(Input/Output) 명세를 제공하여 자동화 실행 기반을 마련하겠습니다. 이는 후크/본문/CTA를 분리하는 원칙을 바탕으로, 모든 산출물이 다음 단계로 매끄럽게 이어지도록 설계되었습니다.

---

## 📝 자동화 워크플로우 데이터 포맷 명세 (Input/Output Specification)

이 명세는 각 에이전트가 생성해야 할 **정확한 구조**를 정의하며, 후속 작업의 오류를 최소화하고 일관성을 보장합니다.

### 1단계: 기획 (Planning)
**목표:** 콘텐츠 방향성 및 경쟁 우위 분석 기반 마련

| 역할 | Input (입력 데이터) | Output (산출물 포맷) | 설명 |
| :--- | :--- | :--- | :--- |
| **Writer / 레오** | 채널 목표, 타겟 청중(학생/직장인), 핵심 주제 (예: To부정사) | **[기획 요약 시트]** | 1. 핵심 메시지 정의 (1줄) 2. 예상 후크 컨셉 (3가지) 3. 경쟁 채널 분석 요약 (강점/약점) 4. 목표 시청 지속률(Retention) 설정 값 |
| **목표** | - | `Planning_Summary.md` | 모든 콘텐츠의 전략적 방향성을 담은 최종 기획 문서. |

### 2단계: 스크립트 (Scripting)
**목표:** 후크, 본문, CTA를 명확히 분리하여 시청 지속률 극대화

| 역할 | Input (입력 데이터) | Output (산출물 포맷) | 설명 |
| :--- | :--- | :--- | :--- |
| **Writer** | 1단계의 `Planning_Summary.md` (특히 채택된 컨셉), 타겟 시청 지속률 목표 | **[스크립트 초안]** | **후크(Hook):** 시청자를 즉시 붙잡는 도입부 (5~10초 분량) **본문(Body):** 핵심 정보 전달 및 논리 전개 (가장 긴 부분) **CTA(Call to Action):** 다음 행동 유도 및 마무리 (구체적 요청 포함) |
| **목표** | - | `Script_Draft_[컨셉명].md` | 후크, 본문, CTA가 명확히 분리된 구조의 상세 스크립트. |

### 3단계: 디자인 (Design)
**목표:** 스크립트의 핵심 메시지를 시각적으로 압축하여 클릭률(CTR) 증대

| 역할 | Input (입력 데이터) | Output (산출물 포맷) | 설명 |
| :--- | :--- | :--- | :--- |
| **Designer** | 2단계의 `Script_Draft_[컨셉명].md` 중 **후크 문구** 및 **핵심 메시지** | **[썸네일 가이드]** | 1. 핵심 후크 문구 (최대 10단어) 2. 시각적 키워드 (Visual Keywords: 감정, 숫자, 대비 강조) 3. 레이아웃 제안 (텍스트 배치 및 색상 분위기) |
| **목표** | - | `Thumbnail_Guide_[컨셉명].txt` | 영상의 후크를 극대화하는 썸네일 디자인 지침서. |

### 4단계: 업로드 (Upload)
**목표:** 검색 최적화(SEO) 및 시청자 유입 극대화

| 역할 | Input (입력 데이터) | Output (산출물 포맷) | 설명 |
| :--- | :--- | :--- | :--- |
| **Writer / 시스템** | 2단계의 최종 스크립트, 3단계의 `Thumbnail_Guide_[컨셉명].txt` | **[메타데이터 세트]** | 1. **제목 (Title):** 후크 + 핵심 키워드 조합 2. **설명 (Description):** 본문 요약 및 관련 해시태그 포함 3. **태그 (Tags):** 주제별, 경쟁사 분석 기반 키워드 목록 |
| **목표** | - | `Upload_Metadata_[컨셉명].json` | 유튜브 업로드에 필요한 모든 최적화된 텍스트 메타데이터 파일. |
