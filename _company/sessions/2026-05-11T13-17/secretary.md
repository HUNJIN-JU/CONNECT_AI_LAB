### 🔗 단계별 데이터 전달 기준 (Data Handoff Criteria)

각 에이전트 간의 효율적인 정보 전달을 위해 아래와 같은 명세에 따라 데이터를 주고받도록 정의했습니다. 이 기준은 자동화 파이프라인의 핵심입니다.

| 단계 | 입력 데이터 (Input) | 출력 데이터 (Output) 및 전달 형식 | 핵심 전달 기준 |
| :--- | :--- | :--- | :--- |
| **1. 기획** | 회사 목표, 이전 의사결정 로그 | `workflow_plan_[컨셉명].json` | 각 단계별 필수 Input/Output 포맷 정의 명시 |
| **2. 스크립트** | 기획 데이터 (Input) | 최종 스크립트 텍스트 + 구조 분석 JSON | 후크, 본문, CTA의 시청 지속률 극대화 전략 반영 여부 |
| **3. 디자인** | 최종 스크립트 (Output) | 디자인 목업 초안 (이미지/PDF 포맷) | 스크립트 내 핵심 메시지를 시각적으로 강조하는 레이아웃 적용 |
| **4. 업로드** | 최종 스크립트 + 디자인 산출물 | `Upload_Metadata_[컨셉명].json` | 영상 제목, 설명, 태그 등 메타데이터의 자동 생성 및 검토 리스트 포함 |
